# v0.1.0 Release Notes

## What This Release Is

v0.1.0 positions repo-health-doctor as a local-first pre-execution safety gate
for AI agents and developers reviewing unknown repositories.

It collects bounded evidence, records limitations, and keeps missing or degraded
visibility from becoming false confidence before repository-derived commands are
run.

## What This Release Is Not

- It is not a replacement for Gitleaks, TruffleHog, OSV-Scanner, Grype, Syft,
  zizmor, actionlint, Falco, Tracee, or an EDR.
- It is not a complete malware sandbox.
- It does not prove that a repository is safe.
- It does not authorize live execution from a gate decision.
- It does not claim third-party security review has happened.

## Stable Contracts

- Default v3 JSON output compatibility remains stable.
- Default CLI behavior remains stable.
- Redaction principles remain stable.
- No finding is not proof of safety.
- Gate decisions and execution authorization remain separate.
- Gate decisions keep `execution_authorized=false`.
- Limitations are gate inputs, not decorative notes.

See [../public-contracts.md](../public-contracts.md).

## Experimental Surfaces

## Known Limitations

- Third-party security review is not done.
- Real compatibility is limited to documented fixture, version, and scope.
- Docker boundary tests in normal CI use a fake/local scanner path.
- Optional real Docker scanner execution depends on the runner environment.
- External adapters remain experimental evidence import paths.
- Current default v3 JSON is check-oriented, not the final formal evidence/gate
  model.

## Third-Party Review Status

Third-party security review is not done and remains external required work.

Use the security model review issue template when requesting external review:

```text
.github/ISSUE_TEMPLATE/security-model-review.yml
```

## Upgrade And Compatibility Note

The initial public release preserves the default v3 JSON report and default CLI
behavior. Opt-in sidecars and experimental schemas are intentionally separate
from that stable path.
