# Docs

詳細ドキュメントを置く directory です。

- `architecture.md`: 設計方針、対象範囲、公開・共有前 preflight としての位置づけ
- `demo.md`: 小さな sample repo を使った実行の流れ
- `rules.md`: JSON report の rule_id、severity、redaction 方針
- `policy.md`: ignore / allow policy の設定方針と `validate-policy` mode
- `release-checklist.md`: 配布前に確認する CLI、CI、redaction、docs 項目
- `../schemas/policy-config.schema.json`: policy config の machine-readable schema
- ルート `README.md`: install、usage、CI 向け quickstart
